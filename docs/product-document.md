# AIMemo 产品文档

记录时间：2026-05-13

## 1. 产品概述

AIMemo 是一个面向个人工作复盘的待办 + AI 周期总结工具。它帮助用户用接近写便签的成本记录任务，用标签和时间组织每日工作，再通过可配置的大模型服务生成日、周、月、年或自定义时间范围的结构化总结。

产品当前以 Flutter 桌面端为主要体验，支持 macOS、Windows、Web 预览、iOS 和 Android 工程；其中 macOS 桌面版使用本地 SQLite 持久化，Web 预览使用内存数据。后端服务已提供邮箱验证码登录、微信小程序登录、云端任务、云端总结历史、免费额度和官方托管模型代理能力。

### 1.1 一句话定位

让个人用户把日常待办自然沉淀为可复盘、可复制、可归档的周期总结。

### 1.2 产品目标

- 降低工作记录成本：用户可以快速添加任务、补充标签和时间，不需要维护复杂项目结构。
- 提升复盘质量：把分散任务转换为结构化总结，减少用户从零组织语言的成本。
- 保护数据控制权：支持本地运行和自定义 OpenAI-compatible 模型服务，真实 API Key 只进入系统安全存储。
- 支持多端延展：通过 AIMemo 账号和后端同步能力，逐步承接桌面端、移动端和微信小程序场景。

### 1.3 非目标

- 不做大型项目管理系统，不承接多人协作、权限审批、甘特图、里程碑等复杂团队管理能力。
- 不做聊天机器人入口，AI 能力围绕任务复盘和总结生成展开。
- 不强制用户使用官方托管模型；自定义模型服务是产品的一等使用方式。
- 不把自定义模型 API Key 写入 SQLite、README、测试、日志或 git。

## 2. 目标用户与场景

### 2.1 目标用户

独立开发者、自由职业者、知识工作者、学生、产品经理、设计师、运营人员，以及任何需要定期整理工作进展和计划的人。

### 2.2 用户痛点

- 每天做了不少事，但到写日报、周报、月报时很难回忆细节。
- 常规待办工具重在“完成任务”，缺少“转化为复盘材料”的能力。
- AI 总结工具需要用户手动整理上下文，容易变成另一个输入负担。
- 用户可能已经有自己的模型服务或本地代理，不希望所有数据都经过第三方托管。

### 2.3 核心使用场景

- 每日记录：工作过程中随手添加任务，完成后勾选或补充内容。
- 标签归类：用标签区分项目、客户、课程、生活或其他主题。
- 周期复盘：在日、周、月、年或自定义时间范围内生成总结。
- 历史回看：查看过去生成的总结，复制到日报、周报、OKR、绩效材料或个人知识库。
- 多端同步：登录 AIMemo 账号后，在桌面端和后续小程序端共享任务与总结数据。

## 3. 产品形态

### 3.1 桌面端

桌面端是当前主要产品形态，采用左右分栏工作区。

- 左侧为任务列表和标签筛选。
- 右侧为操作区，包含添加/编辑任务、生成总结、总结历史。
- 右侧操作区宽度可以拖拽调整，并保存到本地设置。
- 应用首次打开时选择本地运行或登录同步。

### 3.2 移动端

移动端、小程序和 H5 小屏端采用同一套产品架构，不绑定具体 Flutter 实现。页面以顶部主切换组织核心任务：

- 顶部主切换包含“任务”和“总结”两个入口。
- 右上角固定“我的”按钮，进入个人页面。
- 任务页底部放置类似微信聊天输入框的快速添加栏。
- 总结页包含“生成总结”和“历史记录”两个下级 Tab。

移动端底部空间优先留给任务快速输入，保证用户可以像发消息一样随手记录一条任务。

### 3.3 Web 预览

Web 预览用于快速体验和开发验证。当前 Web 预览数据保存在运行时内存中，刷新页面会清空，不作为正式持久化方案。

### 3.4 后端服务

后端位于 `backend/`，承担账号、云端数据、额度和托管模型代理能力。当前默认官方后端地址为：

```text
https://aimemo-backend.onrender.com
```

本地联调时可改为：

```text
http://127.0.0.1:8787
```

## 4. 信息架构

### 4.1 一级模块

- 启动选择：桌面端选择本地运行或登录同步。

- 移动总结栏：包含生成总结和历史记录两个下级 Tab。
- 我的：账号登录、退出、同步状态、免费额度和个人信息。
- 模型设置：配置自定义模型服务或官方模型参数，不承载登录流程。

### 4.2 核心对象

任务：

- 标题
- 正文内容
- 标签
- 完成状态
- 开始时间
- 完成时间

标签：

- 只展示仍有关联非删除任务的标签。
- 最近被任务关联的标签排在前面。

总结：

- 周期类型
- 周期标签
- 周期开始和结束时间
- 标签筛选
- 关联任务
- 模型输出
- 创建时间

模型设置：

- 模型模式：自定义模型或官方托管模型
- 自定义模型 Base URL
- 自定义模型 Model
- API Key 保存状态
- 官方后端地址
- 官方模型可用状态
- 不承载账号登录表单

个人页：

- 当前账号信息
- 登录和退出入口
- 同步状态
- 免费额度
- 后续账号绑定入口

## 5. 核心流程

### 5.1 首次启动

1. 用户打开 AIMemo。
2. 应用展示使用方式选择。
3. 用户选择“本地运行”后直接进入工作区，本机 SQLite 保存数据。
4. 用户选择“登录同步”后输入邮箱并完成验证码登录。
5. 登录同步模式会复用 AIMemo 官方账号会话，启动后自动同步一次，并在运行中周期性同步。

### 5.2 添加任务

1. 桌面端用户进入“添加”面板；移动端用户停留在“任务”栏。
2. 移动端用户在底部聊天式输入框输入一句任务。
3. 点击发送后立即新增任务，开始时间默认为发送时间。
4. 第一版快速添加只处理纯文本，不解析标签或自然语言时间。
5. 用户需要补充标签、开始时间或更多正文时，可以展开详情后进入编辑。
6. 桌面端仍可在添加面板里一次性填写任务内容、标签和开始时间。
7. 任务保存后列表刷新，新标签会出现在标签筛选中。

### 5.3 编辑任务

1. 桌面端用户点击任务列表中的任务，右侧操作区切换到“编辑”状态。
2. 移动端用户点击任务卡片时，只展开或收起任务详情。
3. 移动端用户通过滑动任务卡片露出编辑和删除操作。
4. 进入编辑后，用户可以修改任务内容、标签、开始时间或完成时间。
5. 如果完成时间早于开始时间，保存失败并展示错误。
6. 保存后桌面端继续停留在编辑表单，移动端返回任务列表并刷新详情展示。

### 5.4 完成任务

1. 用户在任务列表中勾选任务。
2. 任务状态变为已完成，并记录完成时间。
3. 再次取消勾选时，任务恢复未完成状态。
4. 列表排序随状态和时间变化更新。

### 5.5 标签筛选

1. 当存在可用标签或已有筛选时，任务列表展示标签筛选条。
2. 用户点击标签后只查看匹配任务。
3. 用户点击“全部”或清除筛选按钮后恢复完整列表。
4. 无非删除任务关联的标签不显示。

### 5.6 生成总结

1. 用户进入“总结”页。
2. 选择模型配置：自定义模型服务或官方托管模型。
3. 选择周期：日、周、月、年或自定义日期范围。
4. 可展开模板编辑器调整总结模板。
5. 可选择标签筛选范围，不选标签表示全部标签。
6. 点击“生成总结”。
7. 应用收集与周期有交集的任务，包括周期前开始但周期内完成或仍未完成的任务。
8. 应用渲染 prompt 并请求模型服务。
9. 总结生成后展示在当前页，并保存到总结历史。

### 5.7 查看总结历史

1. 桌面端用户进入右侧操作区的“历史”页。
2. 移动端用户进入顶部“总结”栏，再切换到“历史记录”下级 Tab。
3. 如果当前为官方托管模型且已登录，优先展示云端总结历史。
4. 否则展示本地 SQLite 总结历史。
5. 用户可以下拉刷新。
6. 用户可以展开一条历史记录查看完整输出。
7. 用户可以复制总结内容。

### 5.8 登录同步

1. 用户在“我的”页完成 AIMemo 账号登录；桌面端可以保留首次启动的登录同步入口，但进入工作区后账号能力统一归口到“我的”页。
2. 模型设置页不再展示登录表单，只根据当前账号状态展示官方模型是否可用。
3. 客户端保存 access token 和 refresh token 到系统安全存储。
4. 桌面端本地任务记录 `clientId`、`cloudId`、`syncStatus` 和 `updatedAt`。
5. 同步时先上传本地新增、更新、软删除，再拉取云端增量变更。
6. 后端通过 `clientId` 支持幂等创建，降低重试造成重复任务的概率。
7. 运行中每 5 分钟自动同步一次。

## 6. 功能规格

### 6.1 任务管理

必须支持：

- 新增任务。
- 编辑任务。
- 勾选完成和取消完成。
- 设置开始时间。
- 编辑时设置或清除完成时间。
- 删除任务，底层使用软删除。
- 按标签筛选。
- 下拉刷新任务列表。
- 移动端点击任务展开详情，再次点击收起详情。
- 移动端滑动任务卡片触发编辑和删除操作。

任务列表排序：

- 未完成任务按开始时间倒序。
- 已完成任务按完成时间倒序。
- 标签排序按最近关联任务靠前。

任务内容规则：

- 任务输入第一行作为列表主标题。
- 后续内容作为任务详情。
- 标签用逗号分隔，并去除空标签。
- 移动端快速添加第一版只按纯文本创建任务，不解析标签或时间。

移动端任务展示规则：

- 未完成任务列表摘要展示开始时间、标签和状态。
- 已完成任务列表摘要不展示开始时间，只展示完成状态、完成时间和标签。
- 展开详情后展示开始时间、完成时间、正文和完整标签。

### 6.2 总结周期

支持周期：

- 日
- 周
- 月
- 年
- 自定义日期范围

周期任务纳入规则：

- 任务开始时间落在周期内，应纳入总结。
- 任务完成时间落在周期内，应纳入总结。
- 任务周期与所选总结周期有交集，应纳入总结。
- 已软删除任务不应作为可见任务参与常规列表和标签展示。

### 6.3 总结模板

默认模板策略：

- 日总结和周总结保持简单，重点是已完成工作和下一步计划。
- 月总结和年总结可以包含更完整的回顾结构。
- 自定义范围在 7 天或以内时按周总结风格处理，超过 7 天时按月总结风格处理。

模板编辑：

- 总结页提供轻量模板模块。
- 模板可展开编辑、保存或重置。
- 如果模板有未保存修改，生成总结前应提示保存。

### 6.4 模型服务

自定义模型：

- 用户配置 OpenAI-compatible 服务。
- 客户端直接请求 `{Base URL}/chat/completions`。
- Base URL 和 Model 可保存到本地 SQLite。
- API Key 保存到系统安全存储。
- 生成总结时才读取真实 API Key。

官方托管模型：

- 用户先在“我的”页使用邮箱验证码登录 AIMemo 账号。
- 客户端调用后端 `/summaries/generate`。
- 后端调用配置好的模型服务。
- 生成成功后扣减免费额度。
- 总结输出、周期、标签和模型元数据保存到云端历史。
- 后端不保存完整 prompt 快照。
- 模型设置页只处理官方模型选择和模型参数展示，不负责登录。

### 6.5 账号与会话

支持：

- 邮箱验证码登录。
- access token 自动刷新。
- refresh token 默认 30 天过期。
- “我的”页统一承载登录、退出、同步状态和免费额度。
- 启动入口“登录同步”和“我的”页复用同一套 AIMemo 官方账号会话。
- 模型设置页读取账号状态，但不提供登录入口。

规划支持：

- 微信小程序登录。
- 邮箱账号与微信账号绑定。
- 多端账号合并。

### 6.6 后端能力

当前后端能力：

- 邮箱验证码登录。
- 微信小程序登录接口。
- 获取当前用户信息。
- 获取免费额度。
- 云端任务 CRUD。
- 云端标签列表。
- 云端总结生成。
- 云端总结历史。
- 客户端配置接口。

主要 API：

- `POST /auth/email/start`
- `POST /auth/email/verify`
- `POST /auth/wechat/mini-program/login`
- `POST /auth/refresh`
- `GET /me`
- `GET /me/quota`
- `GET /tasks`
- `POST /tasks`
- `PATCH /tasks/:id`
- `DELETE /tasks/:id`
- `GET /tags`
- `POST /summaries/generate`
- `GET /summaries`
- `GET /client/config`

## 7. 数据与隐私

### 7.1 本地数据

桌面端本地 SQLite 保存：

- 任务
- 标签派生所需的任务标签字段
- 总结历史
- 总结模板
- 模型基础设置
- UI 设置，例如右侧栏宽度
- 同步元数据

### 7.2 安全存储

系统安全存储保存：

- 自定义模型 API Key
- 官方账号 access token
- 官方账号 refresh token

### 7.3 云端数据

登录同步或官方托管模型场景下，后端保存：

- 用户账号
- 云端任务
- 云端总结历史
- 额度记录
- 刷新会话
- 托管模型调用所需服务端配置

### 7.4 隐私边界

- 自定义模型模式下，任务上下文和 prompt 直接从桌面客户端发送到用户配置的模型服务。
- 官方托管模式下，prompt 会发送到 AIMemo 后端，再由后端调用模型服务。
- 官方托管模式保存总结输出和必要元数据，不保存完整 prompt 快照。
- 真实模型 API Key 不应进入 SQLite、README、`.env` 示例之外的文档正文、测试或 git。

## 8. 界面与交互原则

### 8.1 设计原则

- 优先服务高频记录和复盘，不制造额外配置负担。
- 桌面端保持紧凑、清晰、可扫描。
- 控件尺寸和视觉语言保持一致。
- obvious 控件不额外堆叠说明文字。
- 标签、日期、模型和模板控制都应能被快速理解和点击。

### 8.2 桌面布局

- 左侧任务列表是默认注意力中心。
- 右侧操作区承担当前任务的添加、编辑、总结和历史。
- 任务点击直接进入编辑，不设置额外查看页。
- 右侧操作区宽度可调整，适配不同屏幕和使用习惯。

### 8.3 移动布局

- 顶部使用“任务 / 总结”主切换，保持跨 Flutter App、微信小程序和 H5 小屏端的一致结构。
- 右上角展示“我的”按钮，点击打开全屏个人页面。
- 任务页底部显示聊天式快速添加栏，输入一句任务后发送即新增。
- 快速添加栏只在任务页显示，进入总结页后不占用底部空间。
- 总结页使用“生成总结 / 历史记录”下级 Tab，历史记录不作为移动端一级入口。
- 任务卡片点击展开详情，再次点击收起详情。
- 任务卡片滑动露出编辑和删除操作。
- 已完成任务在列表摘要中隐藏开始时间，展开详情后再展示完整时间信息。
- 任务表单在短屏和键盘弹出时仍可滚动填写。
- 标签筛选和候选标签横向滚动，避免挤压正文。

### 8.4 总结页

- 周期 selector 和日期范围 picker 保持同等高度、边框和可点击感。
- 模板模块默认轻量收起，需要时展开。
- 生成按钮在配置区下方，状态明确。
- 最新总结可以选择和复制。

## 9. 状态与错误处理

### 9.1 空状态

- 无任务时提示用户从当前设备可用入口添加任务。
- 无总结历史时提示生成后会自动保存。
- 无云端总结时提示云端总结会显示在这里。

### 9.2 加载状态

- 任务列表、标签、历史和模型设置使用明确 loading 状态。
- 生成总结时按钮进入“生成中”，避免重复提交。
- 保存模型设置或在“我的”页登录时展示操作中状态。

### 9.3 错误状态

- 任务保存失败展示具体错误。
- 总结生成失败展示模型、网络或配置错误。
- 同步失败用 Snackbar 提示。
- 官方托管模型不可用时，用户侧应看到清晰不可用状态。

## 10. 运营与部署

### 10.1 本地运行

适用于不登录、不同步、使用自定义模型服务的用户。所有任务和总结历史默认保存在本机。

### 10.2 登录同步

适用于需要跨端使用、希望云端保存任务和总结历史的用户。当前桌面端已接入自动同步，后续需要补足冲突可视化、手动重试和大批量分页策略。

### 10.3 Render 部署

仓库根目录提供 `render.yaml`，可以部署后端 Web Service 和 Postgres。免费档适合演示和自测，但不适合长期生产：

- 免费 Web Service 可能休眠。
- 免费 Postgres 有过期限制。
- 邮件验证码建议通过 Resend API 发送。

### 10.4 Windows 安装包

Windows 桌面工程和 Inno Setup 安装脚本已接入。GitHub Actions 的 Windows installer workflow 通过推送 `v*` 版本标签触发。

## 11. 指标建议

产品指标：

- 日新增任务数
- 周活跃记录天数
- 总结生成次数
- 总结复制次数
- 模板编辑次数
- 登录同步启用率
- 自定义模型和官方托管模型使用占比

质量指标：

- 总结生成成功率
- 同步成功率
- 邮箱验证码发送成功率
- access token 自动刷新成功率
- 任务重复创建率
- 客户端启动错误率

隐私与安全指标：

- 真实 API Key 泄漏事件数应为 0。
- 日志中出现敏感 token 或 API Key 的次数应为 0。

## 12. 后续路线

- 增加同步手动重试入口。
- 完善大批量任务分页同步策略。
- 打磨官方托管模型不可用、额度不足和网络失败的提示。

- 基于后端 API 实现微信小程序任务和总结体验。

- 支持邮箱账号与微信账号绑定。
- 支持更多总结历史筛选维度。
- 改善首次登录时本机数据与云端数据合并体验。

长期：

- 形成桌面、移动 App、微信小程序一致的复盘体验。
- 增加按标签、项目和时间维度的复盘资产管理。
