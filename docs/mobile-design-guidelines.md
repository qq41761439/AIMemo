# AIMemo iOS/Android 设计规范 V1

记录时间：2026-05-13

## 1. 定位与范围

AIMemo 移动端先做原生 iOS 和 Android App。iOS 使用 SwiftUI，Android 使用 Kotlin、Jetpack Compose 和 Material 3。两端共享产品结构、设计 token、页面信息架构和验收标准，但控件、导航、返回、弹层、触感反馈和键盘避让遵守各自平台习惯。

V1 聚焦云端账号版：

- 任务
- 标签
- 总结生成
- 总结历史
- 我的

V1 不设计本地运行模式、自定义模型直连、本地 API Key 管理和离线编辑同步。这些能力保留在桌面端或后续版本讨论。

## 2. 设计原则

- 高频记录优先：用户应能像发消息一样快速添加一条任务。
- 复盘配置轻量：周期、日期、标签和模板控制要清晰可点，不堆叠解释文字。
- 工作流安静：界面像任务便签和周期复盘工具，不做营销式大卡片和重装饰。
- 平台原生：共享品牌和结构，但 iOS 看起来像 iOS，Android 看起来像 Android。
- 状态明确：登录、同步、加载、错误、额度不足、生成中、复制成功都必须有可见反馈。
- 可访问性优先：文字可读、触控目标足够大、动态字体可用，不依赖颜色单独表达含义。

## 3. Design Tokens

### 3.1 色彩

| Token | 值 | 用途 |
| --- | --- | --- |
| `color.primary` | `#2F6F5E` | 主操作、选中态、进度、链接、成功 |
| `color.secondary` | `#355C7D` | 总结、历史、信息提示 |
| `color.accent` | `#C7782F` | 额度、未保存、轻提醒 |
| `color.background` | `#F6F7F4` | App 背景 |
| `color.surface` | `#FFFFFF` | 卡片、输入栏、面板 |
| `color.border` | `#DDE2DC` | 描边、分隔线 |
| `color.ink` | `#202622` | 主文字 |
| `color.muted` | `#68716A` | 次级文字 |
| `color.faintText` | `#8A928C` | 弱文字、辅助说明 |
| `color.error` | `#B42318` | 错误、删除 |
| `color.warning` | `#A15C07` | 警告、额度不足 |

深色模式应为单独配色，不做简单反相。品牌主色可以提高亮度和降低饱和度以保证对比度。

### 3.2 间距、圆角和触控

- 间距阶梯：`4 / 8 / 12 / 16 / 20 / 24 / 32`。
- 卡片圆角：`8`。
- 输入框、按钮、选择器圆角：`6`。
- 标签圆角：`999`。
- iOS 最小触控目标：`44pt`。
- Android 最小触控目标：`48dp`。
- 可点击控件之间至少保留 `8` 的间隔。

### 3.3 字体

- iOS 使用 SF Pro 和系统 Dynamic Type。
- Android 使用 Roboto 和 Material 3 type scale。
- 页面标题：`22-24`，字重 `600-700`。
- 分区标题：`16`，字重 `600`。
- 正文：`16`，行高约 `1.45-1.6`。
- 次级文字：`14`。
- 辅助文字、标签：`12`。
- 不使用负字距；列表和按钮文字优先换行，不让文字挤出容器。

## 4. 信息架构

移动端主结构固定为顶部双入口：

- 任务
- 总结

右上角固定“我的”图标按钮，点击进入全屏个人页面。

不使用底部四栏导航。底部空间只在任务页提供聊天式快速添加栏；切到总结页后底部输入栏消失。

总结页内部使用二级 Tab：

- 生成总结
- 历史记录

页面切换必须保留状态：

- 当前标签筛选
- 已展开任务
- 任务列表滚动位置
- 总结周期和日期范围
- 总结历史滚动位置

## 5. 核心页面规范

### 5.1 任务页

任务页由顶部应用栏、标签筛选、任务列表和底部快速输入栏组成。

标签筛选：

- 仅当存在可用标签或已有筛选时显示。
- 横向滚动，不挤压正文。
- “全部”作为第一个选项。
- 选中态使用主色描边或浅主色背景。
- 无非删除任务关联的标签不显示。

任务卡片：

- 点击卡片展开详情，再次点击收起。
- 左侧为完成控件；完成和取消完成都要有即时反馈。
- 滑动卡片露出编辑和删除操作。
- 未完成摘要展示开始时间、标签和未完成状态。
- 已完成摘要隐藏开始时间，只展示完成状态、完成时间和标签。
- 展开后展示开始时间、完成时间、正文和完整标签。
- 卡片内容过长时正文最多先展示数行，展开后完整展示。

底部快速输入栏：

- 固定在任务页底部并避开安全区和键盘。
- 像聊天输入框：单行起步，可扩展到 4 行。
- 输入一句任务后点击发送即创建任务。
- V1 快速添加只处理纯文本，不解析标签或自然语言时间。
- 发送按钮使用图标；发送中禁用并显示进度。
- 为空时发送按钮不可用。

### 5.2 任务编辑

编辑入口可以使用平台 sheet 或全屏编辑页。字段包括：

- 正文
- 标签
- 开始时间
- 完成时间

规则：

- 正文不能为空。
- 标签用逗号分隔，去除空标签。
- 完成时间早于开始时间时阻止保存，并在时间字段附近显示错误。
- 短屏和键盘弹出时表单仍可滚动，保存按钮可达。
- 保存成功后返回任务列表并刷新任务详情展示。

### 5.3 总结页

生成总结页包含周期、日期范围、标签筛选、模板模块、生成按钮和最新总结。

周期与日期：

- 周期 selector 和日期范围 picker 同高、同边框、同圆角。
- 日期范围必须像按钮一样可点击，带日历图标。
- 支持日、周、月、年和自定义日期范围。

模板模块：

- 默认收起，只显示标题、当前状态和展开按钮。
- 展开后可编辑、保存或重置。
- 有未保存修改时，生成总结前提示保存。

生成按钮：

- 位于配置区下方。
- 生成中显示 loading，并禁止重复提交。
- 官方托管模型不可用、未登录、额度不足或网络失败时展示清晰错误。

最新总结：

- 生成后展示在当前页。
- 内容可选择、可复制。
- 复制成功使用短 Toast 或 Snackbar。

### 5.4 历史记录

历史记录是总结页的二级 Tab，不作为移动端一级入口。

列表项展示：

- 周期类型和周期标签
- 标签
- 创建时间
- 模型

点击列表项展开完整输出。无历史时显示“生成后会自动保存到这里。”，不放冗长说明。

### 5.5 我的

“我的”是全屏个人页面，统一承载账号和云端状态。

未登录：

- 展示邮箱验证码登录。
- 邮箱输入使用 email 键盘。
- 发送验证码和登录都有 loading 状态。
- 错误展示在表单附近。

已登录：

- 展示邮箱或账号标识。
- 展示免费额度和官方模型可用状态。
- 展示最近同步状态。
- 提供退出登录入口，退出前确认。

模型设置页不承载登录表单。

## 6. 平台适配

### 6.1 iOS

- 使用 SwiftUI。
- 根结构使用 `NavigationStack`。
- 列表使用 `List` 或自定义列表，滑动操作优先使用系统 swipe actions。
- 编辑任务可使用 `.sheet` 或全屏编辑页。
- 标题、返回、sheet 高度、键盘避让遵守 iOS 习惯。
- 重要确认、完成任务和生成成功可使用轻量 haptic。
- 支持 Dynamic Type，不因文字放大导致按钮文字溢出。

### 6.2 Android

- 使用 Kotlin、Jetpack Compose 和 Material 3。
- 根结构使用 Compose Navigation。
- 顶部栏使用 Material 3 `TopAppBar`。
- 主切换使用 `SegmentedButton` 或同等 Material 3 组件。
- 编辑任务可使用 `ModalBottomSheet` 或独立编辑 destination。
- 反馈使用 `Snackbar`。
- 支持系统返回键：sheet 先关闭，再返回上一页。
- 支持动态字体和深色模式，但不使用系统动态取色覆盖 AIMemo 品牌主色。

## 7. 状态与反馈

所有主要操作必须覆盖：

- idle
- loading
- success
- error
- disabled

错误信息应靠近发生错误的控件。全局错误可以用 Snackbar，但表单验证错误不只放在页面顶部。

任务列表、标签、历史和“我的”页需要明确 loading 状态。加载超过 300ms 时展示骨架或进度，避免空白页。

删除任务必须二次确认或提供撤销入口。删除成功后任务从可见列表移除，标签筛选随任务关联刷新。

## 8. 可访问性与响应式

- 正文对比度至少满足 WCAG AA。
- Icon-only 按钮必须有无障碍标签。
- 不用颜色单独表达错误、成功、选中或警告。
- 支持屏幕阅读器的合理阅读顺序。
- 支持横竖屏和折叠屏窄态。
- 不允许横向滚动页面主体；只有标签筛选、候选标签这类局部控件可以横向滚动。
- 键盘弹出时输入栏、表单字段和保存按钮不能被遮挡。

## 9. 验收清单

设计稿至少覆盖：

- 任务空状态
- 任务列表
- 任务展开
- 任务编辑
- 总结生成
- 总结历史
- 我的未登录
- 我的已登录

尺寸至少覆盖：

- 小屏手机
- 常规手机
- 大屏手机或折叠屏窄态

实现验收：

- 无页面级横向滚动。
- 文字不溢出容器。
- 键盘弹出时输入和保存按钮可用。
- 所有主要操作有 loading、error 和 success 状态。
- iOS 和 Android 都遵守各自平台返回、弹层、触控和字体规则。

## 10. 技术路线确认

- iOS 和 Android 先各自原生实现，不追求代码复用。
- 两端共享设计规范和后端 API 契约。
- Android 优先落地云端账号版。
- 当前 Flutter 桌面端暂时冻结维护，不纳入这份移动端设计规范。
